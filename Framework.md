> **航班驱动下机场陆侧交通中心拥堵溯源与多主体协同调度研究：基于有限容量排队网络和随机滚动优化**

1. 三个问题使用同一套模型体系，避免各问相互割裂。
2. 两座机场共用统一模型结构，仅在参数、瓶颈和调度策略上体现差异。
3. 问题二的调度方案必须由问题一识别出的瓶颈结果推导。
4. 论文中的模型、算法、代码与实验结果必须保持一致。
5. 所有结论均需要数据、仿真或对照实验支撑。
6. 区分拥堵表面现象与真正的因果瓶颈，防止调度方案转移拥堵。
7. 同时考虑旅客体验、驾驶员收益、道路效率与机场管理成本。

## 四、统一模型体系

### 4.1 系统功能区划分

将机场交通中心抽象为有向网络：

\[
G=(V,E),
\]

其中节点和道路包括：

1. 旅客到达节点；
2. 候车节点；
3. 出租车蓄车场；
4. 网约车缓冲区；
5. 车辆入口；
6. 上客泊位区；
7. 私家车即停即走区；
8. 巴士与公交区；
9. 离场道路；
10. 公共道路连接节点。

节点表示服务区域，道路边表示车辆流动路径。节点与道路均具有有限服务能力和存储容量。

### 4.2 航班驱动的客流预测

航班 \(f\) 的旅客离开航站楼时间受到行李提取、步行距离和航站楼组织效率影响。定义总客流到达率：

\[
D_t=\sum_f N_f h(t-T_f),
\]

其中：

- \(T_f\)：航班实际到达时刻；
- \(N_f\)：航班下机旅客人数；
- \(h(\cdot)\)：旅客离开航站楼的时间分布。

可采用对数正态分布、Gamma 分布或多个高斯峰近似旅客离场时间。

交通方式选择采用 Logit 模型：

\[
P_{m,t}
=
\frac{\exp(-\theta C_{m,t})}
{\sum_k\exp(-\theta C_{k,t})},
\]

\[
C_{m,t}
=
\alpha W_{m,t}
+\beta F_m
+\gamma L_m
+\delta R_m,
\]

其中 \(W_{m,t}\)、\(F_m\)、\(L_m\)、\(R_m\) 分别表示等待时间、费用、步行距离和可靠性。

该模型能够描述调度改变等待时间后，旅客主动改变交通方式的反馈行为。

### 4.3 有限容量双边排队网络

对每种交通方式 \(m\)，同时建立旅客队列与车辆队列：

\[
P_{m,t+1}=P_{m,t}+d_{m,t}-x_{m,t},
\]

\[
V_{m,t+1}=V_{m,t}+u_{m,t-\tau_m}-y_{m,t},
\]

其中：

- \(P_{m,t}\)：等待旅客数量；
- \(V_{m,t}\)：待服务车辆数量；
- \(d_{m,t}\)：旅客到达量；
- \(u_{m,t}\)：车辆调入数量；
- \(\tau_m\)：车辆调入延迟；
- \(x_{m,t}\)：完成运输的旅客数；
- \(y_{m,t}\)：完成上客的车辆数。

泊位服务能力约束为：

\[
y_{m,t}\leq \mu_m s_{m,t}\Delta t,
\]

载客能力约束为：

\[
x_{m,t}\leq a_m y_{m,t},
\]

其中 \(a_m\) 为平均载客量，能够正确区分出租车、网约车、私家车和机场巴士。

### 4.4 道路拥堵传播模型

采用元胞传输模型或有限存储容量道路模型描述道路回溢：

\[
n_{e,t+1}=n_{e,t}+f^{in}_{e,t}-f^{out}_{e,t},
\]

\[
f^{out}_{e,t}
=
\min
\left\{
S_{e,t},
R_{e+1,t},
C_{e,t}
\right\},
\]

其中：

- \(n_{e,t}\)：道路 \(e\) 上的车辆数；
- \(S_{e,t}\)：道路发送能力；
- \(R_{e+1,t}\)：下游道路接收能力；
- \(C_{e,t}\)：道路通行能力。

当下游道路存储容量不足时，上游车辆无法继续进入，从而形成：

\[
\text{离场道路拥堵}
\rightarrow
\text{上客区无法释放}
\rightarrow
\text{入口道路排队}
\rightarrow
\text{交通中心整体拥堵}.
\]

## 五、问题一：拥堵瓶颈因果识别

### 5.1 基础诊断指标

计算各节点和道路的：

- 平均容量负荷率；
- 95% 分位负荷率；
- 最大队列长度；
- 平均等待时间；
- 95% 分位等待时间；
- 满载持续时间；
- 排队回溢概率；
- 系统延误贡献率。

### 5.2 表面拥堵指数

构造表面拥堵指数：

\[
S_i
=
0.4\widetilde{\rho_i^{95}}
+0.3\widetilde{Q_i^{\max}}
+0.3P_i^{spill},
\]

其中：

- \(\rho_i^{95}\)：节点负荷率的 95% 分位数；
- \(Q_i^{\max}\)：节点最大队列长度；
- \(P_i^{spill}\)：节点发生回溢的概率；
- \(\widetilde{\cdot}\)：归一化处理。

该指标用于识别拥堵表现最明显的位置。

### 5.3 因果瓶颈弹性

分别将每项设施容量或车辆供给增加 \(\varepsilon=10\%\)，重新运行相同场景。定义因果瓶颈弹性：

\[
E_i
=
-\frac{\Delta J/J}{\Delta C_i/C_i}
=
\frac{J_0-J_i^{+10\%}}{0.1J_0},
\]

其中 \(J\) 为系统广义延误成本：

\[
J
=
\sum_t
\left[
c_pP_t
+c_vV_t
+c_nN_t
+c_bB_t
\right].
\]

其中：

- \(P_t\)：旅客排队量；
- \(V_t\)：车辆排队量；
- \(N_t\)：道路占用量；
- \(B_t\)：受到容量约束的阻塞量。

### 5.4 联合瓶颈判定

| 表面拥堵指数 | 因果瓶颈弹性 | 判定 |
|---|---|---|
| 高 | 高 | 根因瓶颈 |
| 低或中 | 高 | 隐蔽性根因瓶颈 |
| 高 | 低或负 | 表面拥堵，单独扩容可能转移或加剧拥堵 |
| 低 | 低 | 非瓶颈 |

问题一应输出：

- 时序排队演化图；
- 拥堵传播图；
- 因果瓶颈弹性排序图；
- 表面拥堵与因果弹性联合诊断图；
- 两机场瓶颈结果对比表。

## 六、问题二：多主体随机滚动调度

### 6.1 三层调度体系

#### 第一层：需求预测层

每隔 5 分钟更新未来 30 至 60 分钟的：

- 航班到达客流；
- 各交通方式需求；
- 车辆供给量；
- 道路状态；
- 潜在异常风险。

#### 第二层：滚动优化层

主要决策变量：

\[
s_{m,t},\quad
u_{m,t},\quad
r_{m,t},\quad
b_t,\quad
z_{e,m,t},
\]

分别表示：

- 各交通方式可使用的泊位数量；
- 蓄车场车辆释放数量；
- 网约车准入数量；
- 巴士加班数量；
- 动态车道分配方案。

#### 第三层：实时反馈层

实时监测队列长度、道路占用率和等待时间。当指标超过阈值时，触发：

- 减少车辆准入；
- 调整出租车释放率；
- 启用备用泊位；
- 加开机场巴士；
- 引导私家车进入停车楼；
- 启动外部缓冲区等待机制。

### 6.2 优化目标

采用随机模型预测控制：

\[
\min
\left[
\mathbb{E}(J)
+\eta\operatorname{CVaR}_{0.95}(J)
\right],
\]

其中：

\[
J
=
\sum_t
\left(
c_pP_t
+c_vV_t
+c_nn_t
+c_eE_t
+c_s|s_t-s_{t-1}|
\right).
\]

各项分别表示：

- 旅客等待成本；
- 驾驶员等待成本；
- 道路拥堵成本；
- 车辆空驶成本；
- 调度切换成本。

### 6.3 公平性与可实施性约束

旅客服务约束：

\[
W^{95\%}_{passenger}\leq W_{\max}.
\]

驾驶员收益公平约束：

\[
I^{driver}_{Jain}\geq I_{\min}.
\]

泊位数量约束：

\[
s^{\min}_{m}\leq s_{m,t}\leq s^{\max}_{m}.
\]

泊位切换约束：

\[
|s_{m,t}-s_{m,t-1}|\leq \Delta s_m.
\]

车辆释放、道路容量、缓冲区容量以及巴士最低服务频率也需要设置相应约束。

### 6.4 上海虹桥机场差异化调度

虹桥机场采用“精细化动态资源配置与车辆释放控制”：

- 网约车预约时间窗准入；
- 出租车蓄车场批量释放；
- 出租车、网约车与私家车可转换泊位；
- 私家车限时停靠；
- 高峰期动态车道复用；
- 根据离场道路接收能力控制上客车辆释放；
- 利用轨道交通分担异常客流。

核心目标：

> 避免短时间内大量车辆同时进入离场道路，防止拥堵由下游向整个交通中心传播。

### 6.5 杭州萧山机场差异化调度

萧山机场采用“公共交通分流与车辆供给保障”：

- 航班联动巴士加班；
- 夜间出租车最低供给保障；
- 网约车分区预约；
- 调度指令考虑车辆驶入机场的时间延迟；
- 长时间等待的私家车转入停车楼；
- 极端供给不足时启动补贴或跨区域调车。

核心目标：

> 缓解航班集中到达时旅客需求与车辆供给在时间上的错配。

## 七、问题三：鲁棒性与韧性评价

### 7.1 扰动场景

| 场景 | 扰动设置 |
|---|---|
| 航班集中延误 | 客流到达时间整体后移并集中释放 |
| 恶劣天气 | 服务时间增加，道路容量下降 |
| 车辆供给不足 | 出租车与网约车供给下降 20% 至 50% |
| 道路故障 | 关键道路容量下降 30% 至 50% |
| 平台执行失效 | 网约车调度服从率下降 |
| 公共交通中断 | 地铁、公交或巴士运力下降 |
| 需求预测误差 | 实际需求偏离预测值 |

每种场景进行不少于 100 次蒙特卡洛仿真。

### 7.2 对比方案

1. 无调度方案；
2. 固定规则调度；
3. 确定性动态调度；
4. 风险约束动态调度。

### 7.3 鲁棒性指标

主要评价指标包括：

- 旅客平均等待时间；
- 旅客等待时间 95% 分位数；
- 驾驶员平均等待时间；
- 驾驶员单位时间收益；
- 驾驶员收益公平性；
- 系统总延误；
- 最大队列长度；
- 拥堵回溢持续时间；
- 系统恢复时间；
- 调度方案失效概率。

定义系统韧性指标：

\[
R
=
1-
\frac{
\int_{t_0}^{t_1}[J(t)-J_0]dt
}{
(t_1-t_0)J_{\max}
}.
\]

韧性指标越大，表示系统受到扰动后的性能下降越小、恢复速度越快。

## 八、关键实验设计

为形成完整的论文证据链，需要完成以下实验：

1. **瓶颈容量扰动实验**：验证识别出的节点是否为真正瓶颈。
2. **固定方案与动态方案对照实验**：证明动态调度的必要性。
3. **公平约束消融实验**：分析公平约束对系统效率和驾驶员收益的影响。
4. **客流预测消融实验**：评估预测信息对调度效果的贡献。
5. **容量预留消融实验**：验证备用容量对异常场景的作用。
6. **交通方式反馈消融实验**：分析旅客方式选择反馈的重要性。
7. **两机场策略交叉应用实验**：将虹桥方案用于萧山、萧山方案用于虹桥，证明差异化策略的必要性。
8. **参数敏感性分析**：分析模型参数变化对结论稳定性的影响。
9. **蒙特卡洛鲁棒性实验**：评价随机扰动下方案的稳定性与恢复能力。

## 九、论文核心创新点

1. 建立旅客、车辆与道路三者耦合的有限容量排队网络。
2. 使用容量扰动弹性识别拥堵根因，而非仅根据队列长度判断瓶颈。
3. 联合表面拥堵指数与因果瓶颈弹性，识别可能加剧下游压力的错误扩容方案。
4. 将旅客交通方式选择反馈纳入动态调度。
5. 使用公平约束兼顾旅客、驾驶员与机场利益。
6. 采用随机滚动优化，同时提升正常运行效率与异常场景韧性。
7. 使用统一模型解释不同城市机场为何需要不同调度策略。
8. 通过策略交叉应用实验验证差异化调度的合理性。
